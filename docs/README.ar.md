## ⚡ ما هو Qwen Orchestrator؟

**Qwen Orchestrator** هو امتداد قوي مبني حصريًا لـ **[Qwen Code](https://github.com/QwenLM/qwen-code)** — واجهة سطر أوامر البرمجة AI مفتوحة المصدر من Alibaba. يحوّل مساعد برمجة AI واحد إلى **فريق منسق من 22 وكيل AI متخصص** يعملون معًا كقسم تطوير برمجيات احترافي.

### لماذا Qwen Orchestrator؟

تخيل أن لديك فريق هندسة كامل بين يديك: **القائد** الذي ينسق، و**المخطط** الذي يصمم البنية، و**مطورو الواجهة الأمامية والخلفية** الذين يبرمجون بالتوازي، و**المراجع** الذي يتحكم في الجودة، و**مهندس ضمان الجودة** الذي يختبر، و**مهندس الأمن السيبراني** الذي يدقق الثغرات، و**مهندس DevOps** الذي ينشر — و14 متخصصًا إضافيًا، جميعهم منسقون تلقائيًا.

**لا يوفر أي امتداد برمجة AI آخر هذا المستوى من التنسيق المتعدد الوكلاء الاحترافي.**

> ⚠️ **هذا الامتداد يعمل فقط مع [Qwen Code CLI](https://github.com/QwenLM/qwen-code)**. إنه ليس أداة مستقلة، وليس امتداد VS Code، وغير متوافق مع مساعدي البرمجة AI الآخرين. إذا لم يكن Qwen Code مثبتًا لديك، [قم بتثبيته أولاً](https://github.com/QwenLM/qwen-code#installation).

## 🎯 نظرة عامة

يحوّل Qwen Orchestrator نظام Qwen Code إلى **قسم تكنولوجيا معلومات احترافي** مع **22 وكيل AI متخصص**، و**22 مهارة خبيرة**، و**6 أوامر شرطة مائلة**، و**ذاكرة ثابتة (رسم بياني للمعرفة)**، و**خادم أدوات MCP**.

### الضمانات الرئيسية

| الضمان                     | كيف يتحقق                                                                 |
| -------------------------- | ------------------------------------------------------------------------- |
| **بدون نماذج/عناصر مؤقتة** | يُمنع على الوكلاء كتابة أكواد مؤقتة؛ مهارة الأنماط المضادة تكشف المخالفات |
| **بدون قفل على نموذج**     | جميع الوكلاء يستخدمون نموذج المستخدم الافتراضي؛ التجاوز لكل وكيل اختياري  |
| **حماية من الحلقات**       | وكيل المراقبة يكتشف ويكسر حلقات LLM تلقائيًا                              |
| **منع N+1**                | مهارة أفضل ممارسات SQL + وكيل مهندس قواعد البيانات يفرضون التحميل المسبق  |
| **تنسيق SQL**              | كلمات مفتاحية بأحرف كبيرة، عمود واحد لكل سطر، JOINs بمسافة بادئة          |
| **الأمان أولاً**           | الامتثال لـ OWASP Top 10 عبر مهارة تدقيق الأمان وكيل المراجع              |
| **اكتمال CRUD**            | القائد التقني يتحقق أن كل كيان لديه إنشاء/قراءة/تحديث/حذف/استعادة         |
| **دعم متعدد اللغات**       | يدعم TypeScript, PHP, Python, Dart, Rust, Go, Java, C#                    |
| **Context7 اختياري**       | يعمل بدون Context7؛ معه، يحصل الوكلاء على استعلام وثائق في الوقت الفعلي   |
| **استعادة الضغط**          | الحالة تستمر في `.qwen-orchestrator/` — تصمد أمام ضغط نافذة السياق        |

## وضوح المستخدم: اسأل قبل البناء

المنسق **لا يفترض أبدًا** ما تريده. عند بدء مهمة، يستخدم وكلاء القائد والمخطط أداة `AskUserQuestion` لتوضيح المتطلبات قبل كتابة سطر واحد من الكود.

### مثال على التفاعل

```
👤 المستخدم: /orchestrator بناء نظام دفع للتجارة الإلكترونية

🤖 القائد: أحتاج لتوضيح بعض النقاط قبل التخطيط:

   📋 مزود الدفع (السؤال 1/3)
   أي بوابة دفع يجب أن أدمجها؟
   ○ Stripe       — المعيار الصناعي، API ممتاز
   ○ PayPal       — موثوق على نطاق واسع، دعم دولي
   ○ MercadoPago  — الأفضل للأسواق اللاتينية
```

## إعداد النموذج

**بشكل افتراضي، جميع الوكلاء يستخدمون نموذج Qwen Code الافتراضي الخاص بك.** لا يوجد نموذج مضمن — صفر قفل.

```yaml
---
name: commander
# model: أزل التعليق أدناه لتجاوز النموذج الافتراضي للمستخدم
model: qwen-max # ← أزل التعليق وحدد نموذجك المفضل
---
```
